# STEM v1 repo conventions

This repo follows the **STEM v1 standards**. The Standard version is pinned in the top-level `CLAUDE.md` (`**Standard version:**` line) and the contract is the inline copies under `docs/Standards/`. **Read those when a topic falls under their scope** — the rules below are a summary, not a substitute.

## Standards quick map

| Question | Read |
| --- | --- |
| Where does this code go? | `REPO_STRUCTURE.md`, `MODULE_SEPARATION.md` |
| F# or C# for this layer? | `LANGUAGE.md` |
| Why is this Windows-only / how do I keep it cross-platform? | `PORTABILITY.md` |
| Versions, central package management, `.editorconfig` | `BUILD_CONFIG.md` |
| How do I write or run tests? | `TESTING.md` |
| What does CI enforce? | `CI.md` |
| Bumping the standard / migrating | `MIGRATION.md` |

## Hard rules — load-bearing on every turn

- **Banned APIs.** Pure layers (`Core`, `Services`, `Abstractions`, `Protocol`) load `BannedSymbols.txt`. Don't introduce `System.Drawing.*`, `Microsoft.Win32.Registry`, `System.Management`, raw `System.IO.Ports.SerialPort`, or hardcoded paths there. See PORTABILITY for replacements.
- **Composition root only.** Adapter wiring (EF Core, drivers, HTTP clients) lives in the composition root (`<App>.GUI/Composition/` for archetype A; `Stem.<Lib>.DependencyInjection` for B). Never in `Services` / `Protocol`.
- **TFM default is `net10.0`.** `net10.0-windows` is opt-in for Windows-bound drivers and legacy `GUI.Windows` only.
- **One language per project.** Cross-language is via project references, not mixing source files in a single project.
- **CHANGELOG.md updates with every PR.** New entry under `[Unreleased]`.

## When you find drift

Don't editorialise a fix into a *while I'm here* sprawl. Record the drift as a separate `chore:` issue. The structural standards are stable; expansion goes upstream into `llm-settings/shared/standards/`, not into ad-hoc per-repo edits.

## Editing the standards inline

`docs/Standards/*.md` are **regenerated by `apply-repo-standard.ps1`**. Direct edits there are overwritten on next bump. Edit upstream in `llm-settings/shared/standards/`.
